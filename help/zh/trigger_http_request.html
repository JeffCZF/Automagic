HTTP 请求（实验性）

触发器 HTTP 请求（实验性） 在收到使用指定路径和端口的 HTTP 请求时执行。

注意：内置的 Web 服务器既不快，也不应尝试上传大文件。请不要用它来运行网店、企业网站或像 Slashdot 这样的网站。

 

示例：
  • 当请求 URL http://192.168.1.XX:8080/sms?receiver=12345&text=test 时，从 PC 使用手机发送短信。
  • 当请求 URL http://192.168.1.XX:8080/files/xyz 时，返回一个包含 SD 卡上文件列表的页面。
设置
路径
此触发器要处理的路径。支持 glob 模式（*, ?）。
示例：
  • /automagic/test
  • /automagic/files/* 处理所有以 /automagic/files/ 开头的路径
当多个触发器匹配请求的路径时，具有最长路径的触发器优先，因为只能写入一个响应。
示例：当存在路径为 /automagic/test 和 /automagic/* 的两个触发器时，第一个触发器优先。
端口
服务器的端口。
仅绑定到 WiFi 接口
是在所有接口上侦听 HTTP 请求，还是仅在 WiFi 接口上侦听。
保持 WiFi 唤醒
是否保持 WiFi 无线电唤醒，否则 WiFi 无线电可能会关闭以节省电池，从而无法再连接到您的设备。
完成 HTTP 响应
是否通过向客户端发送空响应来完成 HTTP 响应。
变量是一个值的容器，可在许多动作和条件中用于动态定义文本的一部分。
有关说明，请参阅动作脚本。

提供的变量
triggertime
触发器执行的时间
remote_host
远程主机的名称或 IP
remote_port
远程主机的端口
request_path
请求的解码路径（例如 /automagic/test）
method
HTTP 方法（例如 GET, POST, PUT）
header_*
所有标头的值。变量名使用前缀 header_ 和标头名称构建，将短横线替换为下划线并将名称转换为小写。
示例：
  • 标头 Accept 存储在变量 header_accept 中
  • 标头 Accept-Encoding 存储在变量 header_accept_encoding 中
导致无效变量名的标头将被忽略。
param_*
所有请求参数的值。变量名使用前缀 param_ 和参数名称构建，将名称转换为小写。
示例：
  • 参数 test 存储在变量 param_test 中
导致无效变量名的参数将被忽略。
file_*
所有上传文件的路径。变量名使用前缀 file_ 和参数名称构建，将名称转换为小写。
示例：
  • 参数 File 存储在变量 file_file 中
导致无效变量名的参数将被忽略。
http_headers
包含所有接收到的 HTTP 标头的未修改名称和值的映射
http_params
包含所有接收到的 HTTP 参数的未修改名称和值的映射
http_files
包含所有未修改的表单字段名称和上传文件路径的映射
