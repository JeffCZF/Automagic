CDMA 蜂窝基站

触发器 CDMA 蜂窝基站 在设备连接或断开与定义的基站时执行。定义列表内的更改不会调用触发器。

警告：某些设备在屏幕关闭时不会向 Automagic 报告基站信息。

 

示例：
  • 当设备连接到家里的一个基站时，打开 WiFi。
设置
方法
是使用经典方法、现代方法还是两者来查找基站。哪种方法效果最好高度取决于设备。
触发条件
触发器应在连接/断开由基站列表定义的区域时执行流程，还是在每次基站更改时执行（可能很频繁）。
基站列表
基站 ID 的逗号分隔列表。支持 glob 模式（*, ?）。支持变量。
要忽略的基站列表
要忽略的基站 ID 的逗号分隔列表。支持 glob 模式（*, ?）。支持变量。
已连接 / 已断开
触发器应在设备连接还是断开与所列基站的连接时执行。定义列表内的更改不会调用触发器。
唤醒设备
触发器应每隔 X 分钟唤醒设备的 CPU 还是不唤醒。当基站信息未定期更新时，这对于某些设备是必需的。选择更短的时间间隔会消耗更多电池。
打开屏幕
是否应短暂打开显示屏。当基站信息否则不更新时，这对于某些设备是必需的。此选项会增加电池消耗，仅当基站信息否则不更新时才应启用。
强制更新
是否应强制基站更新。此选项会增加电池消耗，仅当基站信息否则不更新时才应启用。
变量是一个值的容器，可在许多动作和条件中用于动态定义文本的一部分。
有关说明，请参阅动作脚本。

提供的变量
triggertime
触发器执行的时间
cell_id
基站 ID，如果未知则为 null
base_station_id
基站的 ID，如果未知则为 -1
network_id
网络的 ID，如果未知则为 -1
system_id
系统的 ID，如果未知则为 -1
base_station_latitude
基站的纬度（四分之一度秒），如果未知则为 2147483647。将值除以 14400 得到十进制度数。并非所有网络都提供有意义的值。
base_station_longitude
基站的经度（四分之一度秒），如果未知则为 2147483647。将值除以 14400 得到十进制度数。并非所有网络都提供有意义的值。
last_cell_id
上一个基站 ID，如果未知或触发器刚刚激活则为 null
last_base_station_id
上一个基站的 ID，如果未知或触发器刚刚激活则为 -1
last_network_id
上一个网络的 ID，如果未知或触发器刚刚激活则为 -1
last_system_id
上一个系统的 ID，如果未知或触发器刚刚激活则为 -1
last_base_station_latitude
上一个基站的纬度（四分之一度秒），如果未知则为 2147483647。将值除以 14400 得到十进制度数。并非所有网络都提供有意义的值。
last_base_station_longitude
上一个基站的经度（四分之一度秒），如果未知则为 2147483647。将值除以 14400 得到十进制度数。并非所有网络都提供有意义的值。
connected
当连接到定义的基站时为 true，否则为 false
disconnected
当与定义的基站断开连接时为 true，否则为 false
