文件观察器

触发器 文件观察器 在文件系统上的感兴趣路径被访问或更改时执行。

 

示例：
  • 在文件被修改后创建其副本。
  • 当文件夹 /sdcard/DCIM/Camera/ 中创建了新的子文件时，将新图像上传到 Dropbox。
设置
要观察的路径
要观察的文件或目录的逗号分隔列表。支持变量（在触发器激活时替换）。
事件类型
要观察的事件类型。
  • 文件已访问：从文件中读取了数据
  • 修改：数据已写入文件
  • 元数据已更改：元数据（权限、所有者、时间戳）被显式更改
  • 可写文件已关闭：有人以写入方式打开了文件或目录，然后将其关闭
  • 不可写文件已关闭：有人以只读方式打开了文件或目录，然后将其关闭
  • 文件已打开：文件或目录已打开
  • 文件已移出：文件或子目录已从受监控的目录中移出
  • 文件已移入：文件或子目录已移至受监控的目录中
  • 子文件已创建：在受监控的目录下创建了新的文件或子目录
  • 子文件已删除：从受监控的目录中删除了文件
  • 删除自身：受监控的文件或目录已被删除；监控实际上停止
  • 自身移动：受监控的文件或目录已被移动；监控继续
变量是一个值的容器，可在许多动作和条件中用于动态定义文本的一部分。
有关说明，请参阅动作脚本。

提供的变量
triggertime
触发器执行的时间
registered_path
受观察的文件或目录的路径
event
以数字表示的事件：
  • 文件已访问：1 (0x00000001)
  • 修改：2 (0x00000002)
  • 元数据已更改：4 (0x00000004)
  • 可写文件已关闭：8 (0x00000008)
  • 不可写文件已关闭：16 (0x00000010)
  • 文件已打开：32 (0x00000020)
  • 文件已移出：64 (0x00000040)
  • 文件已移入：128 (0x00000080)
  • 子文件已创建：256 (0x00000100)
  • 子文件已删除：512 (0x00000200)
  • 删除自身：1024 (0x00000400)
  • 自身移动：2048 (0x00000800)
path
触发事件的文件或目录的路径，相对于主监控的文件或目录
