查询内容提供者

查询内容提供者动作允许从其他应用的内容提供者加载数据。内容提供者将数据以一张或多张表的形式呈现给 Automagic，类似于关系数据库中的表。

警告： 此功能允许访问 Android 未正式提供的数据，可能并非在所有设备型号和 Android 版本上都可用。

 

设置
内容 URI
内容提供者数据的 URI。支持变量。
投影
要返回的表列表。支持变量。
选择条件
声明要返回哪些行的过滤器，格式类似于 SQL WHERE 子句（不包括 WHERE 本身）。选择条件可以包含 ? 字符，这些字符将被内容提供者的 选择参数 中提供的值替换。支持变量。
选择参数
用于替换选择条件中每个 ? 字符的值。支持变量。
排序顺序
如何对行进行排序，格式类似于 SQL ORDER BY 子句（不包括 ORDER BY 本身）。支持变量。
结果类型
定义 Automagic 应如何向流程提供数据：
  • 表格：向流程提供一个行列表，每行包含一个值列表。
  • 表格（转置）：与表格相同但转置。为每一列提供一个包含该列所有值的列表。
  • 列表：将返回的表格转换为扁平列表。
  • 单个值：仅向流程提供返回数据的第一个值。
  • 文本：将返回的表格转换为 CSV 文本。
包含标题行
返回的结果是否应包含带有列标题的行。
变量
用于存储结果的变量名称。支持变量。
变量是存储值的容器，可在许多动作和条件中使用，以动态定义文本的一部分。
参见动作脚本说明。

提供的变量
result
根据 结果类型 定义的结果数据
