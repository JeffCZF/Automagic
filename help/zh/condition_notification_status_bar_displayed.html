状态栏显示通知

条件 状态栏显示通知 检查某个应用的通知是否显示在状态栏中，并将第一个匹配通知的信息作为变量提供给流程。

 

示例：
  • 每 5 分钟检查一次状态栏中是否显示了 Hangout 消息，并播放通知声音。
设置
包名
显示通知的应用的逗号分隔列表。可以使用全局模式来匹配多个包。支持变量。
通知 ID
通知的 ID。支持变量。
过滤
定义必须在可用字段之一中包含的文本。对于 包含文本 和 匹配全局模式，支持变量。
忽略常规事件
定义条件是否应忽略常规事件。
当发布通知的应用未将通知标记为"进行中"时，该通知被视为 常规 通知。
忽略进行中的事件
定义条件是否应忽略进行中的事件。
通知由发布通知的应用程序标记为"进行中"事件，通常在通知显示某些进度时设置，例如文件下载、应用播放声音或应用作为前台服务运行。
忽略组摘要
定义条件是否应忽略组摘要。
组摘要是一个附加通知，用于将多个相关通知分组在一起。
变量是一个值的容器，可在许多动作和条件中用于动态定义文本的一部分。
有关说明，请参阅动作脚本。

提供的变量
title
显示的通知的标题
content_title
通知显示的内容标题
content_title_big
通知显示的大内容标题
notification_text（此功能并非 Android 官方支持，并非适用于所有通知）
通知的文本内容
notification_text_big（此功能并非 Android 官方支持，并非适用于所有通知）
通知的大型展开视图的文本内容
notification_channel_id (Android 8.0+)
通知渠道的 ID
notification_priority (Android 4.1+，此功能并非 Android 官方支持，并非适用于所有通知）
通知的优先级（-2=最低 至 2=最高），当可用时
notification_large_icon（此功能并非 Android 官方支持，并非适用于所有通知）
通知的大图标，当可用时
notification_when
通知中显示的时间
notification_number
通知中显示的数字，当可用时
notification_vibrate_default
通知是否使用默认模式振动
notification_vibrate_pattern
作为数字列表的振动模式（暂停持续时间（毫秒），振动持续时间（毫秒），...），当可用时
notification_sound_default
通知是否应播放默认通知声音
notification_sound_uri
要播放的声音的 URI，当可用时
notification_people (Android 4.4+)
通知相关人员列表（URI 列表），当可用时
notification_info_text (Android 4.4+)
通知显示的信息文本，当可用时
notification_sub_text (Android 4.4+)
通知显示的子文本，当可用时
notification_summary_text (Android 4.4+)
通知显示的摘要文本，当可用时
notification_progress (Android 4.4+)
通知显示的当前进度，当可用时
notification_progress_max (Android 4.4+)
最大进度，当可用时
notification_progress_indeterminate (Android 4.4+)
进度是否不确定，当可用时
notification_text_lines (Android 4.4+)
通知显示的文本行，当可用时
notification_background_image_uri (Android 5+)
要显示的背景图像的 URI，当可用时
notification_category (Android 5+)
通知的类别，当可用时
notification_accent_color (Android 5+)
通知的强调色，当可用时
notification_ticker_text
通知显示的滚动文本
notification_visibility (Android 5+)
通知的可见性（-1=保密，0=私有，1=公开）
notification_ongoing
这是否为进行中的事件
notification_foreground_service
此通知是否用于标记前台服务
notification_local_only
通知是否应仅在本地设备上显示
notification_group (Android 7+)
通知的组密钥
notification_is_group_summary (Android 7+)
通知是否为组摘要
package_name
显示通知的包的名称
app_name
应用的显示名称
id
通知的 ID
buttons（此功能并非 Android 官方支持，并非适用于所有通知）
大型展开通知上可用的按钮列表
wearable_actions (Android 4.4+，此功能并非 Android 官方支持，并非适用于所有通知）
在 Android Wear 上显示的操作列表
count
与条件匹配的通知数量
