UI 事件

触发器 UI 事件 在系统检测到用户界面中的更改时执行流程，例如当窗口打开、按钮被按下、列表被滚动、文本已输入等。

注意：系统不会将所有事件通知给 Automagic。一些发生非常频繁的事件可能会被系统跳过。

 

示例：
  • 当按下 Google Play 中的“安装”按钮时执行流程。
设置
事件类型
  • 窗口已打开：新窗口、对话框或弹出窗口已打开。变量 text 通常包含窗口的标题。
  • 组件已点击：按钮或标签已被点击。图像按钮的变量 text 通常为空。
  • 组件已长按：按钮或标签已被长按。图像按钮的变量 text 通常为空。
  • 组件已选中：一个元素已被选中，例如列表中的一行或下拉菜单中的一个元素。
  • 组件已获得焦点：输入焦点已移动到另一个元素。
  • 组件已滚动：列表已被滚动。
  • 文本已更改：文本输入字段中的文本已更改。
  • 文本选择已更改：文本选择或光标位置已更改。
  • 公告：应用发出公告，例如更改书籍中的页面。

提供的变量
triggertime
触发器执行的时间
event_type
由数字表示的事件：
  • 窗口已打开：32
  • 视图已点击：1
  • 视图已长按：2
  • 视图已选中：4
  • 视图已获得焦点：8
  • 文本已更改：16
  • 文本选择已更改：8192
  • 视图已滚动：4096
  • 公告：16384
content_description
源的内容描述
package_name
发起应用的包名称
source_class_name
源的类名称
source_bounds
元素的位置和大小（无法确定时为 null）
text
源的文本或打开的窗口标题
label
源的标签
enabled
源是否已启用
checked
源是否已选中
fullscreen
源是否在全屏模式下执行
password
源是否为密码字段
scrollable
源是否可滚动
scroll_x
源左边缘的滚动偏移量（像素）
scroll_y
源上边缘的滚动偏移量（像素）
from_index
元素的起始索引
to_index
元素的结束索引
item_count
列表中的元素数量
current_item_index
元素包含在列表中时的项目位置
added_count
编辑文本时添加的字符数
removed_count
编辑文本时删除的字符数
before_text
更改前的文本
app_name
发起应用的名称
