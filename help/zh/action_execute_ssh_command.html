执行 SSH 命令（实验性）

执行 SSH 命令动作连接到 SSH 服务器并执行命令。

注意： 无法保证此操作与所有 SSH 服务器兼容。

 

示例：
  • 每晚在 NAS 上执行命令。
  • 在小工具上显示服务器的可用内存。
设置
服务器/端口
要连接的 SSH 服务器的 IP 或名称。支持变量。
验证主机密钥
是否应检查主机密钥。跳过主机密钥检查会降低 SSH 的安全性。
用户名
用户名。支持变量。
认证类型
使用密码认证还是公钥认证。不支持键盘交互认证。
密码
密码。
私钥文件
私钥文件的路径（OpenSSH 格式）。如果密钥在 Automagic 中无法使用，您可能需要将密钥转换为其他格式。支持变量。
公钥文件
公钥文件的路径（OpenSSH 格式）。如果密钥在 Automagic 中无法使用，您可能需要将密钥转换为其他格式。支持变量。
密码短语
密钥的密码短语，如果不使用密码短语则为空。
测试连接/更新主机密钥
尝试使用给定的凭据连接到指定服务器。变量不会被替换。
命令
要执行的命令。支持变量。
提供输入文件
是否向执行的命令提供输入文件（如 cat file | ssh user@server command）
输入文件
要提供给命令的文件。支持变量。
超时
总超时时间（连接和执行）。
将 stdout 存储在变量或文件中
是否将命令的输出存储在变量中（最大 1MB，文本）。二进制输出应始终存储在文件中。
存储标准输出的变量
命令写入标准输出的输出将存储在此字段定义的变量中。
存储标准输出的文件
用于存储命令输出的文件路径。支持变量。
存储错误输出的变量
命令写入错误输出的输出将存储在此字段定义的变量中。
存储退出代码的变量
命令的退出代码将存储在此字段定义的变量中。
变量是存储值的容器，可在许多动作和条件中使用，以动态定义文本的一部分。
参见动作脚本说明。

提供的变量
stdout
命令的标准输出
stderr
命令的标准错误输出
exit_code
命令的退出代码
