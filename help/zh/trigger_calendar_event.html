日历事件

触发器 日历事件 在日历上的事件开始或结束时执行。

 

示例：
  • 当会议开始时将铃声静音。
  • 在会议开始前 5 分钟振动。
  • 当会议开始时在 Gleeo Time Tracker 中开始记录任务。
设置
在事件开始时触发
触发器是否应在事件开始时执行。持续时间指定触发器应在事件开始之前还是之后的某一时间量执行。
在事件结束时触发
触发器是否应在事件结束时执行。持续时间指定触发器应在事件结束之前还是之后的某一时间量执行。
日历
应检查事件的日历名称的逗号分隔列表。名称可以包含全局模式字符，如 * 和 ?。
标题
要匹配的事件标题的逗号分隔列表。标题可以包含全局模式字符，如 * 和 ?。

示例：
  • Plan* 匹配标题如 Planning、Plan B 等。
描述
要匹配的事件描述的逗号分隔列表。描述可以包含全局模式字符，如 * 和 ?。
位置
要匹配的事件位置的逗号分隔列表。位置可以包含全局模式字符，如 * 和 ?。
事件类型
触发器是考虑全天事件还是常规事件（非全天），或者两者都考虑。
事件可用性
触发器是考虑标记为空闲、忙碌还是暂定的事件。
在设备空闲/打盹中允许 (Android 6+)
定义是否允许此触发器将设备从空闲/打盹模式唤醒。（参见 设备空闲/打盹模式）
像闹钟一样 (Android 5+)
定义此触发器是否应像闹钟一样处理，闹钟通常显示在状态栏中。
此选项确保 Android 在指定时间精确调用触发器，并防止系统延迟事件。
变量是一个值的容器，可在许多动作和条件中用于动态定义文本的一部分。
有关说明，请参阅动作脚本。

提供的变量
triggertime
触发器执行的时间
event_calendar
事件的日历名称
event_title
事件的标题
event_description
事件的描述
event_location
事件的位置
event_start
事件的开始时间
event_end
事件的结束时间
event_all_day
指示事件是否为全天事件的布尔值
event_availability
FREE、BUSY 或 TENTATIVE，取决于事件的可用性
event_id
事件的 ID
