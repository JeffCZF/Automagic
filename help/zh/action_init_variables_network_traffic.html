初始化变量网络流量

初始化变量网络流量动作使用系统或运行特定应用的特定用户 ID 自设备启动以来的发送和接收字节数初始化变量。

注意： 某些设备不收集用户/应用特定值。这也是 Android 4.3 的一个已知问题。

 

示例：
  • 将每天传输的数据量存储到日志文件中。
设置
流量
是存储系统的总体传输数据，还是存储特定应用（或用户 ID）传输的数据。
总发送字节数变量
用于存储自设备启动以来发送字节数的变量名称。
总接收字节数变量
用于存储自设备启动以来接收字节数的变量名称。
移动网络发送字节数变量
用于存储自设备启动以来在移动接口上发送字节数的变量名称。
移动网络接收字节数变量
用于存储自设备启动以来在移动接口上接收字节数的变量名称。
包名
要检查的包名（即运行该应用的用户）。支持变量。
发送字节数变量
用于存储运行指定应用的用户 ID 自设备启动以来发送字节数的变量名称。
接收字节数变量
用于存储运行指定应用的用户 ID 自设备启动以来接收字节数的变量名称。
变量是存储值的容器，可在许多动作和条件中使用，以动态定义文本的一部分。
参见动作脚本说明。

提供的变量
total_transmitted_bytes
发送字节数，如果设备不支持此统计则为 -1
total_received_bytes
接收字节数，如果设备不支持此统计则为 -1
mobile_transmitted_bytes
通过移动接口发送的字节数，如果设备不支持此统计则为 -1
mobile_received_bytes
通过移动接口接收的字节数，如果设备不支持此统计则为 -1
uid_transmitted_bytes
用户 ID 发送的字节数，如果设备不支持此统计则为 -1
uid_received_bytes
用户 ID 接收的字节数，如果设备不支持此统计则为 -1
