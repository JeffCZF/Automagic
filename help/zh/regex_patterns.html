正则表达式

正则表达式可用于测试文本是否包含特定文本，或更一般地检查给定文本是否匹配定义的模式。

基本模式示例：
Test.* 此模式匹配以 Test 开头且后跟任意数量字符的文本，例如 Test、Testabc，但不匹配 abcTest、123Test123
Test.+ 此模式匹配以 Test 开头且后跟任意数量字符，但必须在 Test 之后至少包含一个字符。例如 TestX，但不匹配 Test
[a-c]{3} 此模式匹配恰好由三个 a 到 c 字符组成的文本，例如：aaa, abc, cba，但不匹配 xyz。
\d{3}.* 此模式匹配以三个数字开头的文本，例如 123abc, 123456，但不匹配 12abc 或 abc123。

Automagic 在动作 脚本 或条件 表达式 中通过以下函数支持正则表达式：

  • Boolean matches(String s, String pattern)
    返回字符串 s 是否匹配正则表达式 pattern。
  • Boolean matches(String s, String pattern, List groups)
    返回字符串 s 是否匹配正则表达式 pattern，并将捕获的组填充到现有列表 groups 中。
  • List findAll(String s, String pattern)
    返回 s 中所有匹配正则表达式 pattern 的值的列表。
  • List findAll(String s, String pattern, boolean returnGroups)
    返回 s 中所有匹配正则表达式 pattern 的值的列表。可选地，每个项目由匹配组的列表组成。（参见 正则表达式）
  • String replaceAll(String s, String regex, String replacement)
    通过在字符串 s 中将所有匹配 regex 的子字符串替换为 replacement 来返回修改后的字符串。
  • List split(String s, String pattern)
    使用正则表达式 pattern 作为分隔符，将字符串 s 拆分为字符串列表。


函数 Boolean matches(String s, String pattern) 的示例

  • 测试文本是否包含单词 test：
    result = matches("this is a test", ".*test.*")
  • 测试文本开头是否包含至少三个数字：
    result = matches("1234567", "\\d{3}.*")
    正则表达式本身只需要一个反斜杠，但在 Automagic 的脚本字符串中，一个反斜杠必须再用一个反斜杠转义。

函数 Boolean matches(String s, String pattern, List groups) 的示例

  • 测试文本是否包含数字，并将第一个数字存储在列表 groups 中：
    groups = newList(); result = matches("contact 123456 called", "\\D*(\\d*).*");
    列表 groups 将包含两个元素 "contact 123456 called" 和 "123456"。

函数 List findAll(String s, String pattern) 的示例

  • 返回 s 中所有匹配正则表达式 pattern 的值的列表：
    result = findAll("contact 123456 calls at 8 o'clock", "\\d*");
    列表 groups 将包含两个元素 "123456" 和 "8"。

函数 String replaceAll(String s, String regex, String replacement) 的示例

  • 从文本中删除所有数字：
    result = replaceAll("contact 123456 called", "\\d", "");
    result 将包含 "contact called"。
  • 删除连续的空白字符，并将其替换为一个空格字符：
    result = replaceAll("a b c", "\\s+", " ");
    result 将包含 "a b c"。
  • 用空格分隔文本中的字符：
    result = replaceAll("1234567", "(.)", "$1 ");
    result 将包含 "1 2 3 4 5 6 7 "。
    $1 指代由括号括起来的第一个捕获组中匹配的文本。
  • 在字符之间插入不同的分隔符：
    result = replaceAll("123456", "(.)(.)", "$1/$2-");
    result 将包含 "1/2-3/4-5/6-"。
    $1 指代由括号括起来的第一个捕获组中匹配的文本，$2 指代由括号括起来的第二个捕获组中匹配的文本。

函数 List split(String s, String pattern) 的示例

  • 在每个冒号处拆分文本以创建标记列表：
    result = split("this:is:a:test", ":");
    列表 result 将包含四个元素 "this"、"is"、"a" 和 "test"。
  • 将文本拆分为单词列表：
    result = split("this is a:test", "\\W");
    列表 result 将包含四个元素 "this"、"is"、"a" 和 "test"。

以下文档是正则表达式语法最重要功能的摘录。


转义序列

\ 引用下一个元字符（因此 \. 匹配字面量 .）。
\Q 引用所有后续元字符，直到 \E。
\E 停止引用元字符（由 \Q 开始）。
\\ 一个字面量反斜杠。
\uhhhh Unicode 字符 U+hhhh（十六进制）。
\xhh Unicode 字符 U+00hh（十六进制）。
\cx ASCII 控制字符 ^x（因此 \cH 将是 ^H，U+0008）。
\a ASCII 响铃字符 (U+0007)。
\e ASCII ESC 字符 (U+001b)。
\f ASCII 换页字符 (U+000c)。
\n ASCII 换行字符 (U+000a)。
\r ASCII 回车字符 (U+000d)。
\t ASCII 制表字符 (U+0009)。

字符类

可以使用集合操作构造任意字符类：

[abc] a、b 或 c 中的任意一个。（枚举。）
[a-c] a、b 或 c 中的任意一个。（范围。）
[^abc] 除了 a、b 或 c 之外的任意字符。（取反。）
[[a-f][0-9]] 任意位于任一范围内的字符。（并集。）
[[a-z]&&[jkl]] 任意同时位于两个范围内的字符。（交集。）

大多数情况下，内置字符类更有用：

\d 任意数字字符（请参阅下面的注释）。
\D 任意非数字字符（请参阅下面的注释）。
\s 任意空白字符（请参阅下面的注释）。
\S 任意非空白字符（请参阅下面的注释）。
\w 任意单词字符（请参阅下面的注释）。
\W 任意非单词字符（请参阅下面的注释）。
\p{NAME} 位于给定 NAME 类中的任意字符。
\P{NAME} 不 位于命名类中的任意字符。

请注意，这些内置类不仅仅涵盖传统的 ASCII 范围。例如，\w 等同于字符类 [\p{Ll}\p{Lu}\p{Lt}\p{Lo}\p{Nd}]。 有关更多详细信息，请参阅 Unicode TR-18， 并请记住，每个类中的字符集可能因 Unicode 版本而异。 如果您实际上只想匹配 ASCII 字符，请指定您想要的显式字符； 如果您指的是 0-9，请使用 [0-9] 而不是 \d，后者还会包括古穆基数字等。

量词

量词匹配前一个正则表达式的若干实例。

* 零个或多个。
? 零个或一个。
+ 一个或多个。
{n} 恰好 n 个。
{n,} 至少 n 个。
{n,m} 至少 n 个，但不超过 m 个。

默认情况下，量词是"贪婪的"，意味着它们会匹配尽可能长的输入序列。也有非贪婪量词，它们匹配尽可能短的输入序列。它们与贪婪量词相同，但尾部带有一个 ?：

*? 零个或多个（非贪婪）。
?? 零个或一个（非贪婪）。
+? 一个或多个（非贪婪）。
{n}? 恰好 n 个（非贪婪）。
{n,}? 至少 n 个（非贪婪）。
{n,m}? 至少 n 个，但不超过 m 个（非贪婪）。

默认情况下，量词允许回溯。也有防止回溯的占有量词。它们与贪婪量词相同，但尾部带有一个 +：

*+ 零个或多个（占有）。
?+ 零个或一个（占有）。
++ 一个或多个（占有）。
{n}+ 恰好 n 个（占有）。
{n,}+ 至少 n 个（占有）。
{n,m}+ 至少 n 个，但不超过 m 个（占有）。

零宽断言

^ 在行首。
$ 在行尾。
\A 在输入开头。
\b 在单词边界。
\B 在非单词边界。
\G 在上一个匹配的末尾。
\z 在输入末尾。
\Z 在输入末尾，或在末尾换行符之前。

环视断言

环视断言断言子模式在当前 position 之后（正向预查）或之前（反向预查）匹配（正）或不匹配（负），而不将匹配的文本包含在包含的匹配中。反向预查模式的可能匹配的最大长度不得无限。

(?=a) 零宽正向预查。
(?!a) 零宽负向预查。
(?<=a) 零宽正向后顾。
(?<!a) 零宽负向后顾。

组

(a) 一个捕获组。
(?:a) 一个非捕获组。
(?>a) 一个独立的非捕获组。（子组的第一个匹配是唯一尝试的匹配。）
\n 捕获组 n 已经匹配的文本。

有关如何编号和访问捕获组的详细信息，请参阅 group()。

运算符

ab 表达式 a 后跟表达式 b。
a|b 表达式 a 或表达式 b。

标志

(?dimsux-dimsux:a) 在给定的标志启用/禁用状态下计算表达式 a。
(?dimsux-dimsux) 在给定的标志启用/禁用状态下计算模式的其余部分。

标志有：

i CASE_INSENSITIVE 不区分大小写匹配
d UNIX_LINES 仅接受 '\n' 作为行终止符
m MULTILINE 允许 ^ 和 $ 匹配任意行的开头/结尾
s DOTALL 允许 . 匹配 '\n'（"s" 代表 "单行"）
u UNICODE_CASE 启用 Unicode 大小写折叠
x COMMENTS 允许空白和注释
