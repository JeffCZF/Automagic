执行 Root 命令

执行 Root 命令动作通过将命令传递给 su 以 root 身份执行 shell 命令，并将输出存储在变量中。

警告： 此功能需要 root 权限，不受 Android 官方支持，可能无法在所有设备上正常工作。

 

示例：
  • reboot 重启设备
  • reboot -p 关机
字符串变量在命令传递给解释器之前会被替换。
例如，您可以使用此功能基于当前日期创建文件名：ps > file-{triggertime,datetime,yyyyMMdd}.txt。

命令中的特殊字符必须使用反斜杠转义或适当地放在引号中：
cat 'Test File1.txt' > 'Test File2.txt'
cat Test\ File1.txt > Test\ File2.txt

设置
命令
在此字段中输入要执行的 shell 命令。命令将传递给 su 进行解释。 可用的命令因设备而异。支持变量。
用户
用于执行命令的用户。未指定时默认为 root。支持变量。
SE 上下文
用于执行命令的 SE 上下文。在使用 Android 5+ 或任何其他将 SELinux 设置为 enforcing 的设备时，通常需要此设置。需要允许切换 SE 上下文的 su（如 SuperSU v1.97+）。支持变量。
工作目录
执行命令的路径。支持变量。
超时
命令的超时时间。
存储标准输出的变量
命令写入标准输出的输出将存储在此字段定义的变量中。
存储错误输出的变量
命令写入错误输出的输出将存储在此字段定义的变量中。
存储退出代码的变量
命令的退出代码将存储在此字段定义的变量中。
变量是存储值的容器，可在许多动作和条件中使用，以动态定义文本的一部分。
参见动作脚本说明。

提供的变量
stdout
命令的标准输出
stderr
命令的标准错误输出
exit_code
命令的退出代码
