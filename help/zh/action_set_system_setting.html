设置系统设置

动作 设置系统设置 将指定值存储到系统的设置数据库中。

警告：将系统设置修改为不支持的值可能会损害设备或导致系统崩溃。风险自负！

注意：对某些设置的更改不会立即被系统捕获，需要发生额外事件（例如关闭并重新打开屏幕）才会生效。某些设置需要通过特殊 API 更改，在设置数据库中更改此类设置的值会被系统忽略。

注意：在某些 Android 版本上，可使用 adb 授予权限，从而无需 root 即可修改 secure 和 global 类别下的设置：

  1. 在计算机上安装 adb（搜索互联网了解如何安装 adb）
  2. 在 Android 设备的 系统设置 -> 开发者选项 中启用 USB 调试
  3. 将 Android 设备连接到计算机
  4. 在终端中执行以下命令：adb shell pm grant ch.gridvision.ppam.androidautomagic android.permission.WRITE_SECURE_SETTINGS

  • 更改 Automagic 中常规动作未提供的 ROM 特定设置的值。
在不同设备上有效的设置（并非在所有设备上都可用）
脉冲通知灯/LED
类别： System
名称： notification_light_pulse
值： 0 = 关闭
1 = 开启
夜间模式（Android 7+，在系统界面调谐器中激活后）
类别： Secure
名称： twilight_mode
值： 0 = 关闭
1 = 开启
2 = 自动
位置模式
类别： Secure
名称： location_mode
值： 0 = 位置模式关闭
1 = 仅限设备
2 = 省电
3 = 高精度

或者：
类别： Secure
名称： location_providers_allowed
值： +gps = 打开 GPS
-gps = 关闭 GPS
+network = 打开网络
-network = 关闭网络
警告：每次只能更改一个值！同时更改多个值将导致设置值不一致，只能通过恢复出厂设置来修复。
移动数据
类别： Global
名称： mobile_data
值： 0 = 移动数据关闭
1 = 移动数据开启

设置
类别
  • System：用于常规设置
  • Secure：用于通常无法修改的设置
  • Global：用于对设备所有用户有效的全局设置（Android 4.2+）
名称
要修改的设置的名称。支持变量。
值
要存储在系统设置数据库中的值。支持变量。
