状态栏通知

状态栏通知动作在状态栏上显示一个通知。通知会一直显示，直到通知被点击或按下清除按钮。

 

示例：
  • 当后台流程通过邮件发送您的位置时显示通知。
设置
图标
用于此通知的图标。

从 Android 6 开始支持自定义图标。

注意： Android 通常会将所有非透明部分显示为白色，因此建议创建具有透明背景和白色内容的自定义 PNG 图像。
注意： 选项“自定义图标”可能导致某些 Android 6.0 设备崩溃。
标题
通知的标题。支持变量。
消息
通知的消息。支持变量。
通道（Android 8+）
要使用的通知通道的 ID。支持变量。
一旦创建了通道，应用就无法更改通道设置。使用按钮 通道设置… 在通道创建后更改设置，或在支持的设备上使用动作 修改通知通道 更改设置。
初始通道设置根据通知动作中的设置（如 播放声音、振动、闪烁 LED、优先级 和 可见性）推断得出。
播放声音
勾选时播放默认通知音。
振动
勾选时振动。
闪烁 LED
使用提供的颜色和闪烁率闪烁通知 LED。并非所有设备都支持通知 LED 和用户定义的颜色。
仅本地
是否建议仅在主设备上显示此通知，而不在配对的设备（如 Android Wear 手表）上显示。
强调色
设备在通知中使用的颜色（通常是展开状态栏中图标的背景色）。
持续
将此通知标记为持续事件，并防止从状态栏中删除此通知。选择此选项时必须提供通知 ID。
清除时保留
防止从状态栏中删除此通知。选择此选项时必须提供通知 ID。
通知 ID
定义用作此通知标识的 ID（1 到 999 之间的数字）。支持变量。
当通知用于触发器 状态栏通知被选中 或用于显示类似状态的信息时，此选项很有用。
具有相同 ID 的通知会替换同 ID 的旧通知。可以使用动作 移除状态栏通知 从状态栏中删除通知。

按钮 移除，X 可用于移除状态栏上具有此 ID 的通知。
类别（Android 5+）
通知的类别。类别定义通知是否允许通过中断过滤器。
优先级（Android 4.1+）
定义通知的优先级。低优先级的通知可能仅在展开的状态栏中显示（取决于 Android 版本）。
当优先级低时，系统可能忽略 播放声音 和 振动 等设置；当优先级高时，可能会强制这些设置。

注意： 在 Android 8+ 上，优先级将按如下方式映射到通知通道重要性：Minimum‑>Low，Low‑>Medium，Normal‑>High，High‑>Urgent，Maximum‑>Maximum。
可见性（Android 5+）
通知是秘密、私有还是公共，这定义了通知是否在锁屏上显示。锁屏也必须相应配置。
大消息（Android 4.1+）
定义展开通知时（通过在通知上向下滑动）显示的大消息。当通知作为通知抽屉中最顶部的通知显示时，通常也会显示大消息。您可以通过使用优先级设置来影响通知的顺序。支持变量。
显示进度条
定义是否应在通知中显示进度条。
不确定进度
进度是否应不确定。
进度条值 / 最大值
进度的当前值和最大值（100%）。支持变量。
大图标（Android 3+）
定义是否应在通知中显示大图标。支持变量。
组（Android 7+）
定义此通知的组。支持变量。
组摘要（Android 7+）
定义此通知是否为此组的组摘要。支持变量。
操作（Android 4.1+）
展开通知时（通过在通知上向下滑动），最多三个操作显示为按钮。当通知作为通知抽屉中最顶部的通知显示时，通常也会显示操作按钮。您可以通过使用优先级设置来影响通知的顺序。

提示： 使用动作 执行流程 来执行由多个动作和条件组成的复杂流程。

以下变量将提供给被执行的动作：
  • action_number：包含操作的编号（1、2 或 3）
  • action_text：通知上所选操作按钮的文本标签
Android Wear 的操作（Android 4.1+）
最多三个在 Android Wear 设备上显示的操作。当定义了这样的操作时，其他操作通常不再在 Android Wear 设备上显示。

提示： 使用动作 执行流程 来执行由多个动作和条件组成的复杂流程。

以下变量将提供给被执行的动作：
  • action_number：包含操作的编号（1、2 或 3）
  • action_text：通知上所选操作按钮的文本标签
  • input_value：所选输入值或语音识别的文本
